2.6 TCP Connection Establishment and Termination

Three-Way Handshake


One of the essential question about TCP Three-Way Handshake is: why do we need this? Why not 2 way or 4 way?
The first message is from client to let server know that client wants to establish a connection. It’s a SYN segment
Then server should reply with a ACK message. This message is sent with a server’s SYN segement. After these two steps, client should know that the connection/path to and from server are good to go.
But Server is not sure about this connection since it hasn’t received a reply from client for its SYN segment. Hence, the third segment need to be sent from client to server which is a ACK segment.
After this three-way handshake, the connection is established successfully.

2.7 TIME_WAIT State

MSL (Maximum segment lifetime)： The MSL is the maximum amount of time that any given IP datagram can live in a network.

There are two reasons for the TIME_WAIT state:

  1. To implement TCP’s full-duplex connection termination reliably
  2. To allow old duplicate segments to expire in the network
